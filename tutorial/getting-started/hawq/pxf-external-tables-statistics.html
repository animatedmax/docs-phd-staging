PXF External Tables Statistics - HBase

Overview

The Analyze command ensures that the query planner has up-to-date statistics about a table. If there are no statistics or obsolete statistics, the query planner may make poor decisions during query planning, which can lead to poor query performance on tables with inaccurate or nonexistent statistics.

In this example, you run the analyze command to demonstrate the use of statistics on external tables.

Note: When you enter the SQL queries in this example, make sure you enter the line with the LOCATION keyword on a single line.

  1. Create a dummy text file for this example (This file will contain ten million rows.):

    $ seq 1 10000000 > /tmp/demo.txt

  2. Load this text file into HDFS:

    $ hadoop fs -put /tmp/demo.txt /

  3. You use the psql utility to execute SQL commands that create the database and tables in HAWQ. Start the utility by entering the psql command at the command prompt. For example:

    [gpadmin@pivhdsne sample-data]$ psql
psql (8.2.15)
Type "help" for help.

gpadmin=#

    -

  4. Turn on the timing feature in psql:

    gpadmin=# \timing
Timing is on.
gpadmin=#

    The timing feature displays the time taken to process queries.

  5. Create an PXF external table that points to this text file:

    gpadmin=# drop external table demo;
gpadmin=# CREATE EXTERNAL TABLE demo (val INT)
LOCATION ('pxf://pivhdsne:50070/demo.txt?Fragmenter=com.pivotal.pxf.plugins.hdfs.HdfsDataFragmenter&Analyzer=com.pivotal.pxf.plugins.hdfs.HdfsAnalyzer&Accessor=com.pivotal.pxf.plugins.hdfs.TextFileAccessor&Resolver=com.pivotal.pxf.plugins.hdfs.TextResolver') 
FORMAT 'TEXT' (DELIMITER = '|');

  6. Look at the statistics before you run the analyze command: (default statistics before collecting statistics)

      
gpadmin=#select relpages,reltuples from pg_class where relname='demo';
relpages | reltuples
----------+-----------
1000 |     1e+06
(1 row)
Time: 4.578 ms

    gpadmin=#select val from demo where val=59999;
val 
-------
59999
(1 row)

Time: 6985.484 ms

  7. Run the analyze command on the PXF table to gather statistics:

    gpadmin=#analyze demo;
ANALYZE
Time: 689.777 ms

  8. Look at the statistics after running the analyzecommand:

    gpadmin=#select relpages,reltuples from pg_class where relname='demo';
relpages | reltuples
----------+-----------
4096 |     161858
(1 row)

Time: 1.450 ms

    gpadmin=#select val from demo where val = 59999;
val 
-------
59999
(1 row)

  9. Drop this external table:

    gpadmin=# drop external table demo;

Conclusion

When you use the analyze command, the query planner has statistics available to use for query planning. If the data in a table has been significantly updated, use the analyze command to update the statistics on the table for better query performance.

Notes: By default, PXF assumes that there are 1 million rows in the data source. By analyzing the table, we are giving the HAWQ engine better statistics about the source data, which helps the optimizer to create optimal query execution plans. (The decisions are related to data issues such as whether to redistribute or broadcast, use hash agg or group agg, merge join, hash join, or nested loop, etc.)
